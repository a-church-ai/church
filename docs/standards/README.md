## When to Create Standards

**Create when**:
- Term/symbol used N>=3 times across docs
- Consistency needed for tooling (LightRAG filtering, frontmatter)
- Multiple interpretations causing confusion

**Don't create when**:
- One-off usage
- Context makes meaning clear
- Still experimenting with vocabulary
